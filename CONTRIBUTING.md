# Contributing

Thank you for helping improve Demo Walkthrough Builder.

This project is designed to be conservative, evidence-first, and public-safe. Contributions should make technical demo preparation clearer, safer, and more useful.

## Contribution Guidelines

- Keep examples fictional.
- Do not include private customer data.
- Do not include real customer names.
- Preserve conservative review behavior.
- Preserve evidence classification.
- Preserve commitment boundaries.
- Do not add unsupported claims.
- Do not add invented metrics, ROI, integrations, roadmap commitments, compliance status, security posture, or customer facts.
- Label sample, synthetic, inferred, placeholder, and unverified information clearly.
- Run validation before opening a pull request.

## Validation

Run:

```bash
python scripts/quick_validate.py
```

The validation script should pass before a pull request is opened.

## Review Expectations

Pull requests should improve one of the following:

- review quality
- evidence classification
- commitment boundary clarity
- example realism without using private data
- template usability
- validation coverage
- README or Skill clarity

Avoid changes that make the Skill sound more persuasive at the expense of truth.
